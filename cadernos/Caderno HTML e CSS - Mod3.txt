VIDEO 2 - Git e Github
   
   Versionamento de software - aqui não sera ensinado tudo sobre versionamento. Somente para o uso do curso - umas das funções principais
   (superficialmente) - (O curso em video tem um curso completo de GitHub)
   
   O gerenciador de versão serve para guardar todas as versões de arquivos que foram modificados. Vamos utilizar gerenciador de versão git.
   Ele vai manter versões do seu codigo no seu computador.
   
   No entanto se o hardware der algum problema você perde todos os seus arquivos versionados. Ou se você quer mostrar seu site para alguem
   precisará hospedar em nuvem o versionamento, no caso o GitHub.
   O git vai fazer o versionamento no seu computador, enquanto o GitHub vai hospedar o versionamento na nuvem.
   
   O repositorio é um local onde se guarda coisas. É o local onde ficará o versionamento feito pelo git.
   Enquanto o repositorio remoto é o GitHub. O gitHub é como se fosse um backup do seu repositorio
   
   O GitHub permite trabalhar em equipe, cada um atualiza o seu repositorio local e quando quiser manda para o repositorio remoto.
   Quando se manda do repositorio local para o remoto, se está fazendo um PUSH

VIDEO 5 - 
   
   *da pra deixar o github desktop escuro
   
   Crie uma pasta (repositorio) para servir ao git. Dai clica, no github desktop, "creat a new repository on hard drive"
   o name tem que ser exatamente o nome da pasta. o "local patch" é onde voce vai querer criar o seu repositorio (onde esta a pasta)

VIDEO 6 - 
   
   Quando foir deita uma alteração, o VSCode vai ficar com 1 em baixo da lupa (isso significa que ha uma alteração), tambem  aparece um "M" ao
   lado do nome do arquivo que quer dizer "modificado"
   
   Se voltar no github desktop ele vai perceber e mostrar a alteração. Para que o repositorio do git seja atualizado com a alateração é preciso
   dar "commit". Para isso, no github desktop, no canto inferior esquerdo, dê um titulo e clique em "commit to master". Com isso o repositorio
   local estara atualizado. 
   
   o commit faz a atualização do repositorio local
   
   Para mandar a atualização para o github, no github desktop clique em push "origin", basta entrar no github e ver a atualização
   
   Se voce quiser criar um repositorio de git novo, vai em file, new repository e prencher.
   
   No canto superior esquerdo ele fala qual repositorio esta ativo no momento. "current repository"
   
   Se voce quiser excluir um repositorio, nunca delete a pasta, vai no repositorio corrente que voce quer apagar, va em "repository" e "remove"
   dessa forma voce excluiu o repositorio local, o do github ainda estará la.
   Para deletar do github vai em settings, "delete this repository" (não é reversivel)
   
VIDEO 7 - 
   
   Como botar seu site no ar com o GitHub Pages
   
   acessa o repositorio, (só funciona se o repositorio for publico)
   vai em settings, em github pages, clique em "chose a theme", escolha o minimal e em "select theme", quando voltar, vai ter um link. Ele é
   composto pelo seu NomeDeUsuario.github.io . Ele vai abrir o arquivo readme, na primeira vez pode levar uns 3 minutos pra abrir. Para acessar
   as paginas do projeto coloca uma "/" e o nome da pasta de destino, depois uma "/" e o nome do arquivo.
   
   Depois do "io" voce coloca uma barra e se refere ao caminho desde o repositorio principal ate o arquivo especifico
   O padrão (NomeDeUsuario.github.io) abr o README
   
   ***TESTAR*** => quando o arquivo é index.html não precisa referenciar na URL, só colocar a pasta que ele esta
   
   O README aceita links. Então voce pode criar, na sua pagina principal (README) os links para as paginas especificas que estão dentro do seu
   repositorio (basta escrever a tag "a" como no HTML)
   
   Pra postar seu site, basta criar um repositorio com os arquivos do site e dar PUSH (tem que deixar publico)
   
   Quando voce da commit em um repositorio que nao existe ele vai criar o repositorio. O arquivo HTML deve ter o nome index.html
   
   quando o site estiver PUSHADO vai no github, entra no repositorio, settings, vai em github pages e em source coloca master branch. Depois
   disso la no github pages vai estar o link do site hospedado. Agora voce pode mandar o link do site pra quem voce quiser.
   
   No site qr-code-generator.com é possivel a partir do link do site gerar um QR CODE
   
   O github pages só serve para HTML, CSS e JavaScript (da propria pagina) não tem compatibilidade com banco de dados por exemplo

VIDEO 13 - 
   
   Para colocar imagens de fundo usamos a propriedade "background-image" com o valor "url('NomeDaImagem')
   
   se a imagem for muito pequena ela replicará varias vezes, porem ficara com uma linha separando que é perceptivel quando bem olhado (dependendo
   da imagem) algumas ele vai replicar sem dar essa linha.
   
   Se a imagem for muito grande ela ficará cortada e não aparecerá por completo
   
   Também da pra adicionar uma imagem de fundo no body pra ficar no corpo inteiro do site
   
   É possivel coloca em elementos (como uma div) não só no plano de fundo do site
   
VIDEO 14 - 
   
   *** commit no final do dia de trabalho ***
   
   A propriedade "background-size: 100px 100px" serve para definir a largura e altura da imagem. O primeiro valor é referente a largura e o
   segundo a altura. (da imagem de fundo) se ficar muito pequena, replicarávarias vezes para cobrir toda a area do fundo.
   Ela vai abrir no tamanho padrão se não definir um background-size
   
   A repetição é automatica, mas voce pode usar um "background-repeat: " voce pode usar o valor "no-repeat" para não repetir, "repeat" para
   repetir. Também tem o "repeat-x" para repetir na horizontal e o "repeat-y" para repetir na vertical
   
VIDEO 15 - 
   
   voce pode definir o posicionamento do background com a pripriedade "background-position: " ela pode ter um ou dois valores com 4 posições.
   Ex: background-position: right; (para pegar a parte da direita do fundo) / background-position: left bottom; (vai pegar a parte de baixo a
   esquerda
   
   Se voce quiser centralizar a imagem e ela não centralizar, crie um "height" com o valor 100vh (100% da viewport). Em casos que queira
   centralizar verticalmente e horizontalmente. Ele define quanto % a altura da viewport vai ocupar.
   *viewport = parte branca da pagina web
   
   No "background-position" primeiro vem a coluna e depois vem a linha. Ex: background-position: right bottom
   Serve como um ponto de ancoragem, uma referencia de onde vai ficar o posicionamento da imagem
   
VIDEO 16 - 
   
   O ideal é colocar uma cor de fundo mesmo usando uma imagem. Isso porque a primeira coisa que o navegador carrega é o conteudo, se a rede
   estiver lenta e não carregar a imagem a tempo, o usuario vai ver a cor de fundo em vez da imagem. A cor branca de fundo fica feia de fundo.
   
   O melhor valor para o background-size é o 100%, assim ele fica responsivo e se adaptara, cobrindo toda a viewport, independente do tamanho da
   janela. 
   O problema é que se diminuir muito ele ficara com o fundo aparecendo na vertical em baixo da imagem. Para resolver isso basta colocar
   de valor: 100% 100%. Assim ele ficará com 100% da viewport na horizontal e 100% da viewport da vertical (não esqueça de colocar o height com
   100vh). O problema disso é que, em alguns casos, a imagem ficara achatada e distorcida.
   
   O valor contain (no background-size) deixará a imagem com 100% da proporção original.
   
   Também tem o cover. Ele vai cobrir a tela inteira, mesmo que tenha que dar uma cortada e mostrarsó um pedaço da imagem (mantendo a proporção)
   ele adapta a imagem de acordo com o tamanho da viewport
   
   Tem o auto (automatico) (padrão) que vai deixar o tamanho normal
   
   Se o conteudo for muito maior que o tamanho da imagem de fundo vai ficar com a cor de fundo aparecendo na parte de baixo.
   
VIDEO 17 - 
   
   A propriedade background-attachment
   
   o valor scroll, o fundo vai rolar junto com o conteudo
   
   o valor fixed, o fundo ficará com a imagem fixa (não aumenta ou diminindo se rolar a barra ge rolagem)
   
   A combinação background-size: cover; + background-attachment: fixed; + max-width; => deixa o fundo responsivo sem distorcer o fundo
   
   Existe a shorthand "background" - A ordem da shorthand: color, position, repeat e attachment (o "size" tem que ser definido sozinho)
   
VIDEO 18 - 
   
   O atributo position serve para definir a posição. O valor relativo significa que o position é relativo
   O valor "absolute" deixa o posicionamento absoluto. Podendo configurar as porpriedades "left" e "top" com elas voce define a distancia entre
   a borda lateral e o lado esquerdo e a borda de cima e o lado de cima. Em vez de usar valores absolutos como "100px" podemos usar valores
   relativos como "50%"
   
   A propriedade "transform" serve para transformar. Os valores (o que vai ser transformado) são diversos. O valor translate(); serve para mexer
   em um componente (a mesma função do posicionamento absoluto) nele, se voce colocar (-50%, -50%) junto com os valores de 50% de left e up ele
   vai ficar centralizado exatamente no meio
